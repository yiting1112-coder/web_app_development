# 線上桌遊系統 PRD (Product Requirements Document)

## 1. 專案概述

### 背景與動機
現代人常因為距離、時間或場地限制，無法頻繁與朋友聚在一起玩實體桌遊。為了解決這個問題，並讓桌遊愛好者能夠隨時隨地享受遊戲的樂趣，我們決定開發一款線上桌遊系統，將實體卡牌、資源交換與策略規劃等元素數位化，重現桌上的歡樂時光。

### 目標用戶
- 喜愛桌遊但無法經常與朋友實體聚會的玩家。
- 想要透過網路與遠端朋友互動、進行休閒娛樂的社群用戶。
- 想要嘗試輕度策略遊戲的新手玩家。

### 核心價值主張
提供一個流暢、公平且具備實體核心機制的線上桌遊平台，讓空間距離不再是享受桌遊樂趣的阻礙。

---

## 2. 功能需求

### 主要功能清單
1. **遊戲房間管理系統（建立/加入房間）**
2. **核心回合與狀態管理**
3. **機制：自動發牌與抽牌**
4. **機制：資源交換與談判**
5. **機制：策略規劃與佈局操作**

### 使用者故事 (User Stories)
- **遊戲房間體驗**
  - 作為一名 **玩家**，我希望 **可以建立一個專屬的遊戲房間並獲得邀請碼**，以便 **邀請我的朋友一起遊玩**。
  - 作為一名 **玩家**，我希望 **可以輸入邀請碼加入朋友的房間**，以便 **與他們順利連線進行遊戲**。
- **發牌與遊戲機制**
  - 作為一名 **玩家**，我希望 **系統能在遊戲開始時自動洗牌與發牌**，以便 **確保遊戲的公平性與運作順暢**。
- **資源交換**
  - 作為一名 **玩家**，我希望 **可以提出與其他玩家交換資源的請求（例如用木頭換磚塊）**，以便 **補齊我缺乏的物料來執行我的策略**。
- **策略規劃**
  - 作為一名 **玩家**，我希望 **系統能清晰標示我的資源、手牌以及目前可執行的動作（如建造、打出發展卡等）**，以便 **我可以精確地進行策略規劃**。
- **回合與進度掌握**
  - 作為一名 **玩家**，我希望 **能有對話框與遊戲操作的歷史紀錄**，以便 **在不漏掉資訊的情況下與朋友交流，並隨時掌握遊戲進度**。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask 進行開發與路由控制。
- **前端渲染**：使用 Jinja2 模板引擎處理頁面渲染與資料呈現。
- **資料庫**：使用 SQLite 作為關聯式資料庫，儲存用戶資料、房間資訊與遊戲狀態。

### 效能與安全考量
- **效能**：系統應支援多人同時在線上穩定進行回合切換，確保狀態同步的延遲在可接受範圍內（不影響回合制遊戲體驗）。
- **安全**：
  - 需要基本的登入與身份驗證機制，防止未授權操作。
  - 防作弊機制：敏感的遊戲邏輯（如未揭露的牌庫、判定合法性）均必須在後端驗證，防止前端 API 被篡改。確保不會將其他玩家覆蓋的手牌提前傳送到前端。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須有)
- 使用者註冊與登入。
- 建立遊戲房間與加入房間的機制。
- 核心回合切換機制（判斷輪到誰、結束回合按鈕）。
- 基本的發牌系統、手牌與資源的視覺化呈現。
- 執行簡單策略動作的功能（如消耗資源換取分數或升級）。

### Should Have (應該有)
- 玩家之間的資源交換功能。
- 遊戲內的操作歷史紀錄（Game Log），顯示誰做了什麼動作。

### Nice to Have (加分項)
- 遊戲內的文字對話框（Chat room）。
- 遊戲結算畫面、勝利判定與簡單排行榜。
- 動畫提示與音效，提升遊戲沉浸感。
